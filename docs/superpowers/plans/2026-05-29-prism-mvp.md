# Prism MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. **All work is tracked in beads (`bd`)** ÔÇö each task below maps to a `prism-` issue; set it `in_progress` when you start and `close` it when its gate passes.

**Goal:** Review a local branch/diff using the user's Claude/Codex subscription CLIs at $0, producing structured findings + a markdown report, with optional posting to a GitHub PR.

**Architecture:** One-shot subprocess `Engine`s (claude-cli/codex-cli, API fallback) behind a single interface (ADR-0001/0007). Deterministic orchestrator selects + fans out reviewers; an LLM `Coordinator` judges (ADR-0008). Reviewer prompts are markdown (ADR-0009). Packaged as a Docker image with subscription creds mounted (ADR-0002).

**Tech Stack:** Python 3.12, uv, pydantic, pytest + pytest-mock, ruff, mypy, asyncio, `gh` CLI. No pip/venv. See `docs/superpowers/specs/2026-05-29-prism-mvp-design.md` and `docs/adr/`.

**Conventions for every task:** TDD (write failing test Ôćĺ see it fail Ôćĺ minimal impl Ôćĺ green Ôćĺ commit). Prompts to CLIs go via **stdin, never argv**. No real LLM calls in unit tests ÔÇö fake the subprocess. DRY. Each phase ends on a **gate**: `uv run pytest -m "not live"` green + `uv run ruff check` + `uv run mypy src` clean.

## File structure (locked)

```
prism/
  pyproject.toml                 # uv project; [tool.ruff]/[tool.mypy]/[tool.pytest]
  Dockerfile                     # python+uv+claude+codex+gh+bubblewrap, non-root app user
  bin/prism                      # thin docker-run wrapper (mounts creds + repo)
  prism.example.yaml             # sample config
  agents/
    REVIEWER_SHARED.md           # mandatory rules + finding contract (DRY)
    coordinator.md  security.md  code_quality.md
  src/prism/
    __init__.py
    findings.py                  # pydantic: Severity, Category, Confidence, Finding, ReviewResult, Decision
    config.py                    # typed config + loader
    engines/
      base.py                    # Engine protocol, Effort, ParsedResult, subprocess runner (stdin/JSONL/timeout/retry/heartbeat)
      claude_cli.py  codex_cli.py  api.py  registry.py
    diff/
      source.py                  # DiffSource (git / gh pr diff)
      filter.py                  # noise + generated-marker filter (migration exemption)
    workspace.py                 # .prism/ builder
    prompts.py                   # load+concat REVIEWER_SHARED + agent md + context; sanitize boundary tags
    reviewers/
      registry.py  runner.py     # registry (name->engine/effort/md/schema); runner (prompt->engine->validated findings)
    coordinator.py               # Coordinator interface + FanOutCoordinator (asyncio) + judge pass
    reporter.py                  # ReviewResult -> markdown
    vcs/
      base.py  github.py         # VCSProvider; GitHub via gh (read context, post summary, auth precheck)
    cli.py                       # `prism local` entrypoint
  tests/
    conftest.py                  # fake-subprocess fixture, fixture diffs
    ... mirrors src/ ...
    fixtures/                    # sample diffs + canned reviewer outputs (golden)
    live/test_live.py            # -m live (real CLIs)
```
